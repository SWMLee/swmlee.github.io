红绿色盲+色弱的前端开发者？
发表于 2020-06-01 20:29:34| 更新于 2020-08-30 22:17:22|一般文字杂文随笔
字数总计:1.2k|阅读时长: 3 分钟

  说起来今天是六一儿童节，隔壁同事大老爷们胡子一大把了居然请假过节去了。嗯，对外肯定是要怎么说的。

  说起来，今天起，前门和后门都开放进出，后门不再禁止出入，那后续外出吃饭、购物倒是非常方便了，不用多绕一个多小时了。

  说起来今天又被安排去做新的项目，毕竟原领导不要我们了，随便一丢，就划到其他行政去了，那么帮新朋友在做做旧工作也未必没意义。一拍手，又安排了两个新的前端项目编写。

  说起来今天在看一个报错信息统计图表的折线图时，超出正常值的区域和未超出正常值的线条，颜色居然是一样的，这我就非常纳闷了。按理来说，正常范围的线条是绿色，超出的线条是红色，红绿如此大的差异，我之前应该也是分辨得清楚的。但是我还是忍不住问了一下隔壁同事，果然：“红绿分明，挺明显的啊。”那天中午，我在网上找了些自测，想起来虽然不愿意承认，但结果嘛，还真就是那样。

  我第一次知道自己是色弱应该是初中二年级的时候，那时候学校有安排一次卫生院到校打疫苗，顺便就同时安排了体检。那就有一个本子，花花绿绿五颜六色的斑点，询问上面什么数字、什么图形。那个时候旁边的人跟我的说法不一致，然后有些图案根本看不出什么数字和图形，才知道，哦，原来我是色弱。为什么不是色盲呢？因为中考高考体检的时候，有检查辨色，那就是用的七色纯色图本，我倒是区分出来了的。毕竟纯粹，就是最好的。

  即便大学学的软件工程，我也没有意识到这个问题有什么不足，毕竟虽然高考有些学校、有些专业的确对视力相关类目要求很多，但这都与我不相干，甚至可以自嘲：看起来这辈子都做不了好色之徒了。

  不知不觉，大学毕业之后，就从事了 4 年的 web 相关开发工作。却很少在页面色彩上遇到什么难度。究其原因，一开始是单机模式，前后台一起，好不好看说到底还是我说了算。后来前后台分离了，虽然也是都要搞，框架多起来，那就是别人说了算，因为我们都自说自话，不懂审美，别人怎么弄，就怎么弄。再后来大家想做什么分布式、微服务，就经常拿着图纸去问，这是什么色，这是什么色，突然间发现，原来在我的思想世界里，这是红色、这是蓝色……别人的眼中，这是大红、朱红、嫣红、深红、水红、橘红、杏红、粉红、桃红、玫瑰红、玫瑰茜红、茜素深红、土红……这是普蓝、钴蓝、湖蓝、靛蓝、碧蓝、蔚蓝、宝蓝、藏蓝、黛蓝、孔雀蓝、天蓝、深蓝、淡蓝、瓦蓝、冰蓝、水蓝……所以我根本就不知道那是什么颜色。

  但是目前为止，有影响到我的 web 开发工作了吗？其实没有。为什么呢？因为一开始，我的隔壁同事，他是正常辨色人；后来，就开始找别人的现成；再后来，最多不过是拿到电子档，自己去吸取 RGB 值……嘛，这都是后话了，其实大家还是在追求功能的完善，色彩这样的东西，很多情况下依靠布局、搭配什么的也到底没有那么刺眼了，所以说，即便红绿色盲，又是色弱的，从事 web 开发也没什么问题，毕竟，色彩什么的，只是工作的很小一部分。

  说起来今天被换了行政管理权，被安排到和另一个同事，专门从事前端相关的开发工作，那之前做的中间件、后台、DB、运维什么的，都暂时不做了。哇，一心一意搞前端，看起来以前那些无所谓，似乎都有所谓了。为什么要两个人开发呢？因为没有美工、没有设计，长什么样子就是我们说了算。那我这纯色主义者，岂不是很难展开了？好吧，还有个同事，我写架构、数据交互，他负责显示、布局？

  嗯，说起来，他好像也是个色弱。

  真是个愉快的六一节啊，说到底，差不多是时候重返少年了。
